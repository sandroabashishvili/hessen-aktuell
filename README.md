# Hessen Aktuell

Status: source adapter MVP / publish-ready prototype
Updated: 2026-06-25

Hessen Aktuell is a source-aware regional news portal prototype for Hessen. It generates a static site from whitelisted public sources and keeps source attribution, archive pages, city pages, topic pages, sitemap and diagnostics together.

## Focus

- Hessen regional news portal
- Kassel, Frankfurt, Darmstadt and Wiesbaden
- Politics, transport, police, economy, events and safety
- Static-first publishing for GitHub Pages
- Conservative ingestion: no full-text copying, source links preserved

## Existing output

- Homepage: index.html
- City pages: cities/kassel, cities/frankfurt, cities/darmstadt, cities/wiesbaden
- Topic pages: topics/politics, topics/transport, topics/police, topics/economy, topics/events, topics/safety
- Daily archive: archive/YYYY-MM-DD
- Source data: shared/data/news
- Diagnostics reports: shared/data/diagnostics
- Sitemap and robots.txt generated by the pipeline

## Main commands

Generate local/default build:

    cd /home/sandro/portfolio_projects/hessen_aktuell
    python3 -m shared.py.news_pipeline
    python3 -m shared.py.diagnostics

Generate publish-ready GitHub Pages build:

    cd /home/sandro/portfolio_projects/hessen_aktuell
    HESSEN_AKTUELL_BASE_URL="https://sandroabashishvili.github.io/hessen_aktuell" bash scripts/build_publish.sh

Windows Task Scheduler wrapper:

    scripts/windows_task_update.ps1

## Publish gate

Before publishing, diagnostics should report:

- Overall: ok
- internal links: 0 broken
- sitemap: 0 missing, robots ok
- SEO meta: 0 missing

Transient external source timeouts remain visible in diagnostics, but they do not block publishing when parser output and generated site checks are healthy.

## GitHub Pages

This project includes .nojekyll so GitHub Pages serves folders and assets without Jekyll processing.

Suggested repository name:

    hessen_aktuell

Expected public URL:

    https://sandroabashishvili.github.io/hessen_aktuell

## Automation behavior

scripts/build_publish.sh:

- exports HESSEN_AKTUELL_BASE_URL
- runs the news pipeline
- runs diagnostics
- if the folder is a git repository and AUTO_COMMIT=1 is set, commits and pushes generated changes
- if the folder is not a git repository yet, only generates and checks the site

scripts/windows_task_update.ps1:

- runs the build script through WSL
- writes logs under shared/data/automation_logs
- is intended for Windows Task Scheduler
